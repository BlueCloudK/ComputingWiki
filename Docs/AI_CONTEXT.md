# AI_CONTEXT

## Purpose

ComputingWiki is an Obsidian-style knowledge graph for computing, software engineering, IT systems, and practical engineering work.

The repo is intended to be useful for both humans and AI agents:

- Humans use it to navigate concepts, learning routes, and project concerns.
- AI agents use it to extend the graph consistently without guessing the structure again.

## What this repo is

- A computing/software/IT knowledge graph.
- A project-oriented reference map.
- A set of concise concept nodes connected by controlled wikilinks.
- A foundation for future expansion packs.

## What this repo is not

- It is not a dictionary of isolated definitions.
- It is not a tutorial collection.
- It is not a place for long code examples.
- It is not a dump of raw source material.

## Language and naming

- Canonical titles and filenames use English.
- Main explanations use Vietnamese.
- Source trace stays in English.
- Aliases can include English and Vietnamese terms.
- Keywords are plain text search terms.

## Node expectation

Each concept node should explain the concept through:

- context
- boundary
- core mechanism
- project role
- artifacts
- decision checklist
- failure modes
- over-engineering warning
- direct child/component links
- strong related links with reasons
- broad related terms as plain text
- search keywords
- short source trace

## Graph discipline

Wikilinks must be controlled. A link should help navigation or reasoning, not just mark that two words are related.

Use wikilinks for direct child/component relationships and strong relations. Use plain text for broad associations, keywords, source names, and future concepts that do not exist yet.
