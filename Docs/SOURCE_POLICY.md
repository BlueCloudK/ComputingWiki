# SOURCE_POLICY

## Purpose

This repo should keep public knowledge organization, not copyrighted raw source material.

## What can be committed

- MOC files.
- Node files.
- Path files.
- Public project docs.
- Short source traces.
- Self-written summaries.
- Source index files that point to official pages.

## What should not be committed

- Raw PDFs.
- Ebooks.
- Downloaded books.
- Large copied extracts.
- Private local source folders.
- Symlinks that expose local private paths.

## Source trace rules

Source trace should be short and in English.

Good examples:

- `Kubernetes official docs`
- `MDN Web Docs`
- `OWASP Cheat Sheet Series`
- `Designing Data-Intensive Applications`
- `ACM/IEEE Computing Curricula 2020`

Avoid long bibliographic blocks inside every node. Use concise trace names so readers and AI agents can understand the basis without turning nodes into citation pages.

## Copyright discipline

- Do not copy long text from books, PDFs, or articles.
- Prefer self-written summaries.
- Use direct quotes only when necessary and keep them short.
- Do not commit raw copyrighted sources.

## Source preference

Prefer official and high-quality sources:

- official docs
- standards
- body of knowledge documents
- textbooks
- classic references
- respected engineering guides

Avoid random SEO tutorials as a primary source.
