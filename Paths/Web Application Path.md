# Web Application Path

## Mục tiêu

Path này giúp người đọc đi từ nền tảng browser tới xây web app có UI, state, API call, performance và security cơ bản.

## Khi nào dùng path này

- Khi học hoặc xây frontend/web application.
- Khi debug lỗi render, layout, request hoặc cookie/session.
- Khi cần nối frontend với backend API.

## 1. Nền tảng web

- [[HTML]] — cấu trúc nội dung của trang.
- [[CSS]] — style, layout và responsive behavior.
- [[JavaScript]] — logic chạy trong browser.
- [[Browser]] — runtime thật của web app.
- [[DOM]] — cây object mà JavaScript thao tác để đổi UI.

## 2. Render và layout

- [[Browser Rendering]] — hiểu browser biến code thành pixels thế nào.
- [[Rendering Pipeline]] — parse, style, layout, paint và composite.
- [[CSS Box Model]] — nền để debug size, spacing và overflow.
- [[Flexbox]] — layout một chiều thường dùng trong UI.
- [[CSS Grid]] — layout hai chiều cho page hoặc vùng phức tạp.
- [[Responsive Design]] — giúp UI chạy trên nhiều viewport.

## 3. Runtime và interaction

- [[Event Loop]] — hiểu async, callback và UI responsiveness.
- [[Web API]] — các capability browser cung cấp cho JavaScript.
- [[Web Storage]] — lưu dữ liệu nhỏ ở browser.
- [[Form Validation]] — kiểm tra input trước khi gửi.
- [[Client Side Routing]] — điều hướng trong SPA không reload trang.

## 4. Gọi backend

- [[HTTP Client]] — lớp gửi request từ frontend.
- [[Fetch API]] — API browser phổ biến để gọi HTTP.
- [[API Contract]] — giúp frontend/backend đồng thuận schema.
- [[CORS]] — boundary security giữa origin.
- [[Cookie]] — state/session đi cùng request browser.

## 5. App architecture

- [[Frontend Framework]] — tổ chức UI bằng component/state.
- [[SPA]] — web app render phần lớn trên client.
- [[SSR]] — render HTML từ server để cải thiện load/SEO/UX.
- [[CSR]] — render nội dung chính trên client.
- [[Hydration]] — gắn JavaScript behavior vào HTML server-rendered.

## 6. Performance và accessibility

- [[Core Web Vitals]] — metric chính của UX web.
- [[Web Performance Metric]] — đo tốc độ và độ ổn định của UI.
- [[Code Splitting]] — giảm bundle ban đầu.
- [[Web Accessibility]] — giúp app dùng được với nhiều người và thiết bị.
- [[Lighthouse]] — audit nhanh performance/accessibility/best practices.
