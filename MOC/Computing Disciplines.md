# Computing Disciplines

Aliases: computing fields, ngành computing

Type: Computer Foundation

## Bản chất

Computing Disciplines là vùng phân biệt các nhánh lớn như computer science, software engineering, information systems, cybersecurity và data science. Nó giúp hiểu một vấn đề thuộc loại kiến thức nào để chọn cách học/tra cứu đúng.

## Dùng trong dự án để làm gì

Computing Disciplines là MOC điều hướng nền tảng, dùng khi bạn cần đặt một khái niệm vào đúng vùng tri thức trước khi đào sâu. Trong dự án, nó giúp tránh lẫn giữa vấn đề thuật toán, hệ thống, bảo mật, dữ liệu và vận hành.

## Khi nào cần quan tâm

- Bạn không chắc vấn đề thuộc CS, SE, Security, IT hay Data Science
- Cần chọn nguồn học hoặc map kiến thức phù hợp
- Một quyết định dự án chạm nhiều discipline
- Muốn tổ chức graph theo vùng tri thức lớn

## Output / artifact nên có

- Decision/routing note về discipline liên quan
- Danh sách node con cần đọc tiếp theo
- Map học tập hoặc project knowledge area

## Checklist kiểm tra

- Vấn đề này thuộc discipline nào là chính?
- Có discipline phụ nào ảnh hưởng quyết định không?
- Node con nào giúp xử lý vấn đề thực tế trước?
- Có đang học nhầm vùng quá rộng so với nhu cầu dự án không?
- Cần chuyển sang MOC chuyên sâu nào tiếp theo?

## Lỗi / rủi ro thường gặp

- Lẫn discipline làm chọn sai nguồn hoặc sai cách giải
- Đọc quá rộng mà không gắn với vấn đề dự án
- Nhầm nền tảng học thuật với checklist triển khai
- Graph bị rối vì vùng lớn link xuống quá nhiều node nhỏ

## Khi nào chưa cần hoặc dễ over-engineer

- Chưa cần phân loại kỹ nếu đang xử lý bug cụ thể đã rõ domain
- Dễ over-engineer nếu dùng taxonomy thay cho việc giải quyết vấn đề trước mắt
